## 2024-05-24 - Regex Match Counting Performance
**Learning:** For counting regex matches in Python, `len(re.findall(...))` is measurably faster (about 15-20% speedup) than `sum(1 for _ in re.finditer(...))` because `findall` executes entirely at the C-level, avoiding the Python interpreter overhead of generator expression evaluation. This is specifically relevant in performance-critical text processing where memory limits are not strict (as `findall` materializes the entire list in memory).
**Action:** When only the count of occurrences is needed and input strings are bounded (like email bodies or headers), use `len(re.findall(...))` to skip Python-level iteration overhead.
